# Heiman Home для HomeAssistant

---
![logo@2x.png](custom_components/heiman_home/brand/logo%402x.png)
## Обзор

Эта интеграция Home Assistant позволяет использовать умные устройства Heiman в системе умного дома. Она подключается к облачному API Heiman и использует протокол MQTT для обновления статуса устройств в реальном времени.

С помощью этой интеграции пользователи могут легко интегрировать устройства Heiman в Home Assistant и использовать их для различных автоматизаций и мониторинга, включая управление прошивкой и контроль дочерних устройств.

## Функции

- 🔌 **Поддержка нескольких устройств**: Шлюзы, датчики, переключатели, сигнализации и многое другое
- ☁️ **Интеграция с облаком**: Подключение через аккаунт Heiman с аутентификацией OAuth2
- 📡 **Обновления MQTT в реальном времени**: Мгновенные обновления статуса устройств через push MQTT
- 🔄 **Управление прошивкой**: Проверка и установка обновлений прошивки прямо из Home Assistant
- 👨‍👩‍👧‍👦 **Управление дочерними устройствами**: Добавление, удаление и управление подчиненными устройствами шлюза через MQTT
- 🏠 **Поддержка нескольких домов**: Независимое управление несколькими домами Heiman
- 🎛️ **Комплексные сущности**: Датчики, переключатели, кнопки, селекторы, бинарные датчики и сущности обновления
- ⚙️ **Настройка через веб-интерфейс**: Простая настройка без конфигурации YAML

<a name="installation"></a>
## Установка


### Способ 1: HACS (Рекомендуется)

#### Первая установка
1. Откройте **HACS** → **Интеграции**
2. Нажмите **+ ИССЛЕДОВАТЬ И ЗАГРУЗИТЬ РЕПОЗИТОРИИ**
3. Найдите `Heiman Home` или нажмите на три точки (⋮) → **Пользовательские репозитории**
4. Добавить репозиторий: `https://github.com/hass-user/heiman-home` с категорией `Integration`
5. Нажмите **ЗАГРУЗИТЬ ЭТОТ РЕПОЗИТОРИЙ**

#### Обновление компонента
1. Откройте **HACS** → **Интеграции**
   ![Download (1)](https://github.com/Elwinmage/ha-xsense-component/assets/15807572/3220c686-f53f-4766-9523-e3272a6ff104)
2. Найдите **Heiman Home**
   ![img.png](image/img.png)
3. **Нажмите **ОБНОВИТЬ** или **Загрузить снова****
   ![img_1.png](image/img_1.png)
### Способ 2: Ручная установка через Samba/SFTP

1. Загрузите последнюю версию с [GitHub Releases](https://github.com/hass-user/heiman-home/releases)
2. Распакуйте папку `heiman_home`
3. Скопируйте папку `heiman_home` в директорию `custom_components` Home Assistant
   ```
   /config/custom_components/heiman_home/
   ```
![img_2.png](image/img_2.png)
![img_3.png](image/img_3.png)
### Способ 3: One-Key-Shell через SSH/Terminal и SSH Add-on

```shell
wget -O - https://raw.githubusercontent.com/hass-user/heiman-home/main/install.sh | bash -
```

### После установки

1. Перезапустите Home Assistant
2. Перейдите в **Настройки** → **Устройства и службы**
3. Нажмите **+ Добавить интеграцию**
4. Найдите `Heiman Home`

<a name="configuration"></a>
## Конфигурация

### Добавление интеграции через веб-интерфейс

1. Откройте веб-интерфейс Home Assistant
2. Перейдите в **Настройки** → **Устройства и службы**
3. Нажмите **Добавить интеграцию**
4. Найдите `Heiman Home` и выберите его
5. Авторизуйтесь с помощью аккаунта Heiman (OAuth2)
6. Выберите дом, который хотите интегрировать
7. Завершите настройку

### Аутентификация

Интеграция использует OAuth2 для безопасной аутентификации:
- Нажмите **Авторизовать**, чтобы войти в аккаунт Heiman
- Предоставьте необходимые разрешения
- Выберите дом, который хотите интегрировать

### Несколько домов

Вы можете добавить несколько домов Heiman:
1. Каждый дом создает отдельную запись конфигурации
2. Каждый дом имеет независимое управление устройствами
3. Настройте каждый дом отдельно в **Устройства и службы**

### Фильтрация устройств

Вы можете фильтровать, какие устройства должны быть интегрированы:
1. Откройте настройки интеграции
2. Перейдите в **Параметры**
3. Включите/отключите определенные устройства
4. Сохраните изменения

### Проверка обновлений

### Установка обновлений прошивки

3. Нажмите кнопку **Установить**
4. Отслеживайте прогресс обновления в реальном времени
5. Устройство будет перезапущено после завершения обновления

### Функции обновления
- ✅ Автоматическое обнаружение версии
- ✅ Мониторинг прогресса (0-100%)

- ✅ Сравнение версий
- ✅ Пакетная проверка обновлений

<a name="mqtt-integration"></a>
## Интеграция MQTT

### Обновления в реальном времени
Интеграция использует MQTT для мгновенных обновлений статуса устройств:
- Без задержек опроса
- Мгновенные изменения статуса
- Меньшее использование API
- Лучшая производительность

### Конфигурация MQTT
- Автоматически настраивается во время установки
- Использует брокер MQTT Heiman
- Безопасное соединение с TLS/SSL
- Не требуется ручная настройка

### Поддерживаемые функции MQTT
- Обновления свойств устройств
- Статус онлайн/офлайн
- События сигнализации
- Показания датчиков
- Управление дочерними устройствами (регистрация, отмена регистрации, обнаружение)

<a name="child-device-management"></a>
## Управление дочерними устройствами

Интеграция предлагает комплексное управление дочерними устройствами через библиотеку `heiman-connect`.
Вся логика управления устройствами реализована в SDK, а интеграция Home Assistant предоставляет простой API для доступа к этим функциям.

### Использование менеджера дочерних устройств

```python
from heimanconnect import ChildDeviceManager

# Получить менеджер дочерних устройств из API клиента
device_manager = await api_client.async_get_child_device_manager(
    user_id="your_user_id",
    devices=devices_dict,
    user_display_name="Your Name"
)

# Добавить дочернее устройство (рекомендуемый метод)
result = await device_manager.add_and_sync_device(
    gateway_product_id="1733421468953686016",
    gateway_device_id="1760910156165971969",
    child_device_id="1768080341172985856",
    child_product_id="1734821218500292608",
    child_device_name="Door Sensor"
)

# Удалить дочернее устройство (рекомендуемый метод)
result = await device_manager.remove_and_sync_device(
    gateway_product_id="1733421468953686016",
    gateway_device_id="1760910156165971969",
    child_device_id="1768080341172985856",
    product_id="1734821218500292608",
    device_name="01000053"
)
```

### Доступные методы

Для подробной документации всех доступных методов и их параметров см. [документацию библиотеки heiman-connect](https://pypi.org/project/heiman-connect/).

<a name="advanced-configuration"></a>
## Расширенная конфигурация

### Конфигурация логгера

Включите отладочное логирование для устранения неполадок:

```yaml
# configuration.yaml
logger:
  default: warning
  logs:
    custom_components.heiman_home: debug
    heimanconnect: debug
```

### Настройка сущностей

```yaml
# configuration.yaml
homeassistant:
  customize: !include customize.yaml

# customize.yaml
sensor.heiman_temperature:
  friendly_name: "Living Room Temperature"
  device_class: temperature
  unit_of_measurement: "°C"

binary_sensor.heiman_door:
  friendly_name: "Front Door"
  device_class: door
```

### Исключение атрибутов

```yaml
# configuration.yaml
heiman_home:
  exclude_attributes:
    - raw_data
    - firmware_info
    - configuration
```

<a name="services"></a>
## Сервисы

### `heiman_home.refresh_device`

Вручную обновить статус устройства.

```yaml
service: heiman_home.refresh_device
data:
  device_id: "1972646416676724736"
```

### `heiman_home.refresh_all_devices`

Обновить все устройства в интеграции.

```yaml
service: heiman_home.refresh_all_devices
```

<a name="debugging"></a>
## Отладка

### Получение атрибутов статуса сущности

1. Откройте **Инструменты разработчика** → **Состояния**
2. Найдите свою сущность (например, `sensor.heiman_temperature`)
3. Просмотрите все атрибуты и текущее состояние

### Получение журналов отладки

1. Включите отладочное логирование (см. [Конфигурация логгера](#advanced-configuration))
2. Откройте **Настройки** → **Система** → **Журналы**
3. Найдите `heiman_home` или `heimanconnect`

### Частые проблемы

#### Устройство не отображается
- Проверьте настройки фильтрации устройств
- Убедитесь, что устройство онлайн в приложении Heiman
- Перезапустите интеграцию

#### Обновление прошивки не работает
- Убедитесь, что устройство онлайн
- Проверьте совместимость устройства
- Убедитесь, что обновление прошивки доступно в приложении Heiman

#### Проблемы с подключением
- Проверьте подключение к интернету
- Проверьте учетные данные аккаунта Heiman
- Проверьте журналы Home Assistant на наличие ошибок

#### MQTT не подключается
- Убедитесь, что сеть разрешает исходящие соединения на `spmqtt.heiman.cn:1884`
- Проверьте настройки брандмауэра
- Перезапустите Home Assistant

<a name="troubleshooting"></a>
## Устранение неполадок

### Аутентификация не удалась
1. Повторно авторизуйте интеграцию
2. Проверьте учетные данные аккаунта Heiman
3. Убедитесь, что аккаунт имеет доступ к выбранному дому

### Устройства не обновляются
1. Проверьте статус подключения MQTT в журналах
2. Убедитесь, что устройство онлайн
3. Попробуйте ручное обновление через сервис

### Высокое использование базы данных
- Исключите ненужные атрибуты (см. [Исключение атрибутов](#advanced-configuration))
- Отключите неиспользуемые сущности
- Проверьте сущности со слишком большим количеством изменений состояния

### Проблемы с производительностью
- Уменьшите интервал обновления, если возможно
- Отфильтруйте неиспользуемые устройства
- Отключите MQTT, если он не нужен (не рекомендуется)
